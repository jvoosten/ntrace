Detailed Description

Instance of central logging manager.

Central logging manager; there is only ever one instance of this object (it's a singleton). It will accept messages and distribute them to the output objects.

Note
Important note: the manager is thread-safe, and uses threads to handle writing messages to the output(s). This is to minimize impact on execution speed of the main program while some output may take a while to process.
