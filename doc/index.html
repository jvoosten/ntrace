Introduction

Debugging your program, or simply keeping track of what it is doing is never easy. printf() statements scattered throughout the code clutter up your source files and either produce too much or too little output. Hooking up the program to a debugger can show great detail of what the program is doing and why, but doesn't present a clear picture of the program flow.

NTrace is a small library that, in combination with some clever C-style macros, helps to track program flow and show only the debug information you are interested in. It is certainly not the first, the last or the best debug library, but it might just be what you are looking for.

Features

  • printf() style formatting
  • Uses C preprocessor; little to no overhead for production version
  • Adjustable debug levels per module
  • Tracks function enter and leaves
  • Millisecond timestamping with process and thread ID
  • Output can be redirected to multiple targets
  • Thread-safe

However, there are a few limitations:

  • Works with C++ only
  • Although NTrace is thread-safe, there is no guarantee messages will be written in order.

Basic functionality

There are two parts in NTrace:

  1. A way to keep track of function calls as the program enter and leaves them;
  2. Print debug info using the printf() syntax.

The latter has a debug 'level', so that more or less information can be printed as desired. This level is set per module and can be changed at runtime (modules are described below).

The NTrace library consists of just a few C++ classes; after installation there will be a shared library and some C++ header files on your system. This header file also contains macros that provide all logging functionality for NTrace; you are however free to create your own macros.

Concepts

NTrace uses the concept of input and output channels. An input channel generates log messages and an output channel takes those log messages and stores them somewhere, with optional formatting. In the middle is a Manager which keeps track of input and output channels and queues log messages.

There can be multiple input and output channels but there is only one manager.
