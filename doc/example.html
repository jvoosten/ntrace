Examples

Okay, so let's get right into it. Let's assume we are writing a Mandelbrot fractal program and we want to trace what it is doing. A typical piece of code may then look like this:

#define ENABLE_NTRACE #include "ntrace.h" #include "mandelbrot.h" TR_MODULE("Mandelbrot"); MandelBrot::MandelBrot (float minx, float maxx, float miny, float maxy) { TR_FUNC("[%f,%f; %f,%f]", minx, miny, maxx, maxy); .... value = new int[lines * columns]; if (!value) { TR_ERR ("Could not allocate memory!"); } .... } void MandelBrot::calculateAll () { TR_FUNC (); for (int i = 0; i < lines; i++) { TR (NTrace::Info, "line = %d", i); calculateLine (i); } } void MandelBrot::calculateLine (int row) { TR_FUNC (); for (int j = 0; j < columns; j++) { TR (NTrace::Debug, "column = %d", j); value[row][j] = calculatePixel (row, j); } }

In the first lines we define ENABLE_NTRACE that enables some of the macros that are used further in the code. If we do not enable this most of the macros evaluate to no-ops (e.g. they vanish from compilation). Next we include the main header file, ntrace.h

TipRather than setting ENABLE_TRACE in every file either set it globally in your compiler settings for your project, or even better, create your own header file and enable it there, based on your own debug preferences. You can also add extra macros there for your convenience.

Example output

( 6209) >> Mandelbrot::Mandelbrot(float, float, float, float) ([-2.000000, -1.500000; 1.000000, 1.500000]) ( 6209) << Mandelbrot::Mandelbrot(float, float, float, float) ( 6209) >> void Mandelbrot::calculateAll() ( 6209) line = 0 ( 6209) >> void Mandelbrot::calculateLine(int) ( 6209) << void Mandelbrot::calculateLine(int) ( 6209) line = 1 ( 6209) >> void Mandelbrot::calculateLine(int) ( 6209) << void Mandelbrot::calculateLine(int) ( 6209) line = 2 ( 6209) >> void Mandelbrot::calculateLine(int) .... ( 6209) line = 638 ( 6209) >> void Mandelbrot::calculateLine(int) ( 6209) << void Mandelbrot::calculateLine(int) ( 6209) line = 639 ( 6209) >> void Mandelbrot::calculateLine(int) ( 6209) << void Mandelbrot::calculateLine(int) ( 6209) << void Mandelbrot::calculateAll()

The first column shows the process ID; then if a function is being entered (>>) or left (<<), together with the function name and parameter types. Any additional parameters that are declared in the TR_FUNC macro are shown as well. Any deeper function calls are indented, which makes it easy to see where each function call starts and ends.

In this example you do see the TR(1,...) call in calculateAll, but not TR(2,...) in calculateLine. That is because the default debug level is 1 for each module. More about this on the macros page.
